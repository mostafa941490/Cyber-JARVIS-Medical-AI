## 🚀 طريقة التشغيل السريعة

### الطريقة الأولى: استخدام السكريبت التلقائي

#### على Windows:

#### على Mac/Linux:

- **زر "API Mode"** = يستخدم Python KNN
- **زر "Local Mode"** = يستخدم خوارزمية JavaScript محلية

### 3️⃣ أدخل بيانات المريض
- اسم المريض (اختياري)
- عدد مرات الحمل

- ضغط الدم
- سُمك الجلد
- مستوى الإنسولين
- مؤشر كتلة الجسم (BMI)
- نسبة الوراثة
- العمر

### 4️⃣ اضغط "فحص الحالة"
- سيظهر رسالة "جاري المسح..."
- سيتصل بـ Python API
- ستظهر النتائج في:
  - مؤشر نسبة الخطورة
  - المؤشرات الدائرية للصحة
  - سجل النظام بالتفصيل

## 📊 فهم النتائج

### نسبة الخطورة (Risk Percentage)
- **0-30%**: ✅ حالة طبيعية (أخضر)
- **30-60%**: ⚠️ خطر متوسط (أصفر/برتقالي)
- **60-100%**: 🚨 خطر مرتفع (أحمر/وردي)

### المؤشرات الصحية
كل مؤشر دائري يعرض نسبة قرب القيمة من المستوى المثالي:
- **80-100%**: قراءة ممتازة (أخضر)
- **60-80%**: قراءة جيدة (أصفر)
- **40-60%**: قراءة متوسطة (برتقالي)
- **0-40%**: قراءة تحتاج انتباه (وردي/أحمر)

## ⚙️ إعدادات متقدمة

### تغيير منفذ API
في `api.py` السطر الأخير:
```python
app.run(host='0.0.0.0', port=5000, debug=True)
# غيّر 5000 إلى أي منفذ آخر
```

ثم في React، عدّل `src/app/components/MedicalDashboardWithAPI.tsx`:
```typescript
const API_URL = "http://localhost:5000"; // غيّر المنفذ هنا
```

---

### استخدام بياناتك الخاصة
ضع ملف `diabetes.csv` في مجلد `python_api` بالصيغة التالية:
```
Pregnancies,Glucose,BloodPressure,SkinThickness,Insulin,BMI,DiabetesPedigreeFunction,Age,Outcome
6,148,72,35,0,33.6,0.627,50,1
1,85,66,29,0,26.6,0.351,31,0
...
```

---

### تحسين دقة الموديل
في `model.py` يمكنك تعديل:
```python
model = KNeighborsClassifier(n_neighbors=7)
# جرّب قيم مختلفة: 3, 5, 7, 9, 11
```

أو استخدم موديل آخر:
```python
from sklearn.ensemble import RandomForestClassifier
model = RandomForestClassifier(n_estimators=100)
```

## ❓ حل المشاكل الشائعة

### 🔴 المشكلة: "ModuleNotFoundError: No module named 'flask'"
**الحل:**
```bash
pip install flask flask-cors pandas scikit-learn numpy
```

---

### 🔴 المشكلة: "FileNotFoundError: diabetes.csv"
**الحل:**
- الكود سيستخدم تلقائياً بيانات تجريبية مدمجة
- أو استخدم `diabetes_sample.csv`
- أو أضف ملف `diabetes.csv` الخاص بك

---

### 🔴 المشكلة: "Address already in use - Port 5000"
**الحل 1:** أوقف البرنامج الذي يستخدم المنفذ 5000
**الحل 2:** غيّر المنفذ في `api.py`

---

### 🔴 المشكلة: واجهة React لا تتصل بـ API
**الأسباب المحتملة:**
1. Python API غير مشغّل
2. منفذ خاطئ في الكود
3. مشكلة CORS

**الحل:**
1. تأكد أن `python api.py` يعمل
2. تحقق من Console في المتصفح للأخطاء
3. استخدم "Local Mode" كحل بديل مؤقت

---

### 🔴 المشكلة: نتائج غير دقيقة
**الحل:**
1. تأكد من صحة البيانات المدخلة
2. استخدم موديل مدرب على بيانات أكثر
3. جرّب تعديل معاملات الموديل
4. استخدم موديلات أخرى (Random Forest, SVM, Neural Networks)

## 🎯 نصائح الاستخدام

### للمطورين
- ✅ استخدم `debug=True` في `api.py` للتطوير
- ✅ راقب Console في Terminal لمتابعة الطلبات
- ✅ استخدم `test_api.py` لاختبار جميع الوظائف

### للإنتاج
- ⚠️ غيّر `debug=False` في `api.py`
- ⚠️ استخدم خادم إنتاج (Gunicorn, uWSGI)
- ⚠️ أضف مصادقة (Authentication)
- ⚠️ استخدم HTTPS
- ⚠️ ضع حدود للطلبات (Rate Limiting)
